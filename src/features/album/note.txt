実装メモ:
🌟アルバムに関するメモ
  1つのプリに保存する画像は複数 → Blobの配列 & インデックスとは別のテーブルで保存 (DB設計.txt)
  画像のプレビューは横にスクロール
  ℹ️削除ボタン → 警告プロンプト(いいえ / 削除)
  ℹ️アルバムのインポートプロンプト
    読み込みエラー (アラート)
    古いアルバムと新しいアルバムを結合する → しない / 結合
      (する場合)   重複するプリを削除 → キャンセル / 削除
      (しない場合) 古いアルバムを削除して新しいアルバムで置き換え → (手の施しようがないから)キャンセル / 置き換え
  ℹ️アルバムのエクスポートプロンプト
  　プロンプト(n枚のプリを含むアルバムをエクスポートします(30秒ほどかかることがあります) → キャンセル / 続行)

アルバム機能のDB設計:

インデックステーブル:
🚫id - プリの識別ID(UUID, 文字列型)
☑timestamp - 保存した日時(UNIX時間, 数値型)
☑favorite - お気に入り登録(ブール値)
☑location - 保存した場所(緯度・経度, 数値型の配列)
☑tags - タグ(文字列の配列)

画像テーブル:
🚫id - プリの識別ID(結合キー)
☑images - 画像の配列(Blob型)
